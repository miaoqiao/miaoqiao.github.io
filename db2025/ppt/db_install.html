Database Installation and Usage

This specification aims at providing information about two full-featured DBMSs, SQLite and DuckDB, that can be useful for you in the future.

SQLite

You will first need to install SQLite on your development machine.

  • Ubuntu Linux or Windows: Please follow the instructions.
  • Mac OS X: On Mac OS Leopard or later, you don't have to! It comes pre-installed. You can upgrade it, if you absolutely need to, with Homebrew.

Check if sqlite3 is properly working by following this tutorial.

DuckDB

Please follow the instructions to install DuckDB for the command line environment.

  1. To load the dataset into DuckDB, we use the SQLite Scanner extension in DuckDB. Install the extension on the DuckDB terminal: 
    			$ duckdb                         
			v1.1.3 19864453f7
			Enter ".help" for usage hints.
			Connected to a transient in-memory database.
			Use ".open FILENAME" to reopen on a persistent database.
			D INSTALL sqlite_scanner;

The small University Database used in our demo

Please copy the DDL file and SQL code for creating small relations from the textbook page. Use sqlite to create an database by loading the DDL and SQL code with the following command in the same directory of the two downloaded files:

                sqlite3 university
                sqlite> .read DDL.sql
                sqlite> .read smallRelationsInsertFile.sql
                sqlite> .tables

You will see a file called university, which is the database of university. To load the university database using either SQLite or DuckDB: 

                $ sqlite3 university
		SQLite version 3.40.1 2022-12-28 14:03:47
		Enter ".help" for usage hints.
		sqlite>
or 
                $ sqlite3
		SQLite version 3.40.1 2022-12-28 14:03:47
		Enter ".help" for usage hints.
		Connected to a transient in-memory database.
		Use ".open FILENAME" to reopen on a persistent database.
		sqlite> .open university
		sqlite>
